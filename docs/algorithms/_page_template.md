## アルゴリズムページの2類型

対象アルゴリズムは次のいずれかに当てはまる。以降のスケルトンは共通だが、
型に応じて各節の要否・中身が変わる（分岐は各見出しの直後にコメントで示す）。

**最適化アルゴリズム型**（EGO, NSGA-II, ParEGO等 — これまでの大多数）
  探索アルゴリズム全体、または探索ループ内の意思決定機構を指す。
  「saealibでの構成」節でコンポーネント組み合わせ表＋実行可能な完全コードを示す。

**ユーティリティ系アルゴリズム型**（Hypervolume計算, 非優越ソート等）
  単一の計算処理を指す。「saealibでの構成」に相当する節を「saealibでのAPI」に置き換え、
  組み合わせ表ではなく関数シグネチャと呼び出し例を示す。「計算量」節が必須になる。

## 掲載範囲の判断

`.claude/exp_ref/named_algorithms_component_map.md` に、既存アルゴリズムのsaealib上の
再現度が ✅Direct（厳密に一致）/ ⚠️Approx（近似）/ ❌Bug（既知バグで再現不可）で
分類されている。❌Bugのアルゴリズムはバグ修正まで掲載しない。⚠️Approxは掲載してよいが、
「近似の場合の注記」節を必ず設ける。

## 擬似コード・フローチャートの書式

擬似コードは`sphinx-proof`の`{prf:algorithm}`ディレクティブを使う。
ラベルは`alg-<アルゴリズム名のスラッグ>`（例: `alg-ego`）で統一する。

```{prf:algorithm} <アルゴリズム名>
:label: alg-<slug>

**Inputs** ...
**Output** ...

1. ...
2. ...
```

フローチャートは`sphinxcontrib-mermaid`を使う。
components/index.mdのパイプライン全体図と同じ`flowchart`記法に揃え、
擬似コードの各ステップ番号がsaealibのどのステージ・コンポーネントに対応するかを示す
（次の「saealibでの構成」節への橋渡しにする）。
